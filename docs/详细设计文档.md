# 贪吃蛇游戏详细设计文档

## 1. 项目目标

本项目实现一个基于 Python 的桌面版贪吃蛇游戏，要求具备以下特点：

- 使用 Python 标准库实现，避免额外依赖
- 提供可运行的图形界面
- 核心逻辑可测试、可维护
- 具备基础的异常场景处理和文档说明

## 2. 总体架构

项目采用分层设计，分为以下两层：

### 2.1 游戏引擎层

文件：`snake_game/engine.py`

职责：

- 管理蛇的位置、方向、食物和分数
- 处理移动、吃食物、碰撞检测、胜利检测
- 对外提供纯逻辑接口，便于测试

### 2.2 界面层

文件：`snake_game/app.py`

职责：

- 使用 `tkinter` 绘制棋盘、蛇和食物
- 接收键盘输入
- 控制游戏刷新节奏
- 展示状态信息，如分数、暂停、结束和胜利

## 3. 模块设计

## 3.1 `GameConfig`

用于管理游戏配置参数：

- `width`：地图宽度
- `height`：地图高度
- `initial_length`：蛇的初始长度
- `random_seed`：随机种子，方便测试复现

## 3.2 `SnakeGame`

核心数据：

- `snake`：按顺序保存蛇身体坐标，`snake[0]` 为蛇头
- `direction`：当前移动方向
- `food`：当前食物坐标
- `score`：当前得分
- `game_over`：是否结束
- `won`：是否胜利

核心方法：

- `reset()`：重置游戏状态
- `set_direction()`：设置移动方向，并阻止直接反向移动
- `change_direction_by_key()`：将按键映射为方向
- `tick()`：执行一个游戏时钟周期

## 4. 关键流程设计

### 4.1 初始化流程

1. 创建 `SnakeGame`
2. 根据配置生成初始蛇身
3. 随机生成不与蛇重叠的食物
4. 将状态渲染到界面

### 4.2 移动流程

1. 根据当前方向计算新的蛇头坐标
2. 判断是否即将吃到食物
3. 检测墙体碰撞和自身碰撞
4. 未碰撞则更新蛇身
5. 如果吃到食物则加分并刷新食物
6. 如果没有空余格子则判定胜利

### 4.3 输入控制流程

1. 监听键盘事件
2. 若为方向键，则更新方向
3. 若为 `Space`，切换暂停状态
4. 若为 `R`，重置游戏

## 5. 碰撞与边界策略

本项目采用以下规则：

- 撞墙即失败
- 撞到自身身体即失败
- 不允许直接朝相反方向移动，避免一步撞回自己
- 当蛇未增长时，允许下一步移动到当前尾巴所在位置，因为尾巴会同步离开

## 6. 测试设计

测试文件：`tests/test_engine.py`

覆盖场景：

- 初始化是否正确
- 普通移动是否正确
- 吃食物后长度和分数是否正确
- 非法反向移动是否被阻止
- 合法转向是否生效
- 撞墙是否结束
- 撞自己是否结束
- 移动到旧尾巴位置是否合法
- 棋盘填满后是否正确判定胜利
- 重置后状态是否恢复
- 非法配置是否抛出异常

## 7. 可维护性设计

- 核心逻辑与界面完全分离
- 通过类型注解提升可读性
- 使用数据类管理配置和状态
- 测试以功能行为为中心，便于回归验证
