# 贪吃蛇游戏详细设计文档

## 1. 项目目标

本项目实现一个基于 Python 的贪吃蛇游戏，要求具备以下特点：

- 使用 Python 标准库实现，避免额外依赖
- 提供可运行的终端界面
- 提供产品化浏览器图形界面
- 核心逻辑可测试、可维护
- 提供分数、等级、最高分和难度递增机制
- 提供开始菜单、难度选择、本地排行榜和音效反馈
- 具备基础的异常场景处理和文档说明

## 2. 总体架构

项目采用分层设计，分为以下三层：

### 2.1 游戏引擎层

文件：`snake_game/engine.py`

职责：

- 管理蛇的位置、方向、食物和分数
- 处理移动、吃食物、碰撞检测、胜利检测
- 根据吃到的食物数量计算等级和移动速度
- 输出 `TickResult` 供界面层驱动动画和状态反馈
- 对外提供纯逻辑接口，便于测试

### 2.2 界面层

文件：`snake_game/app.py`

职责：

- 启动本地 HTTP 服务和浏览器页面
- 提供游戏状态查询和输入控制接口
- 使用浏览器 Canvas 渲染棋盘、蛇和食物
- 展示 HUD、状态覆盖层、开始菜单、难度卡片、等级提示和粒子动画
- 使用 Web Audio 在浏览器侧合成轻量音效

### 2.3 终端界面层

文件：`snake_game/terminal_app.py`

职责：

- 使用 `curses` 在终端中绘制棋盘
- 接收方向键和 `WASD` 输入
- 在终端环境下提供稳定可用的运行方式
- 提供分数、等级、速度和状态栏
- 提供纯文本渲染函数，方便测试

### 2.4 数据持久化层

文件：`snake_game/storage.py`

职责：

- 负责最高分的本地持久化读写
- 负责排行榜记录和排序
- 屏蔽文件不存在、内容损坏等异常场景

## 3. 模块设计

## 3.1 `GameConfig`

用于管理游戏配置参数：

- `width`：地图宽度
- `height`：地图高度
- `initial_length`：蛇的初始长度
- `random_seed`：随机种子，方便测试复现
- `points_per_food`：每个食物提供的分数
- `foods_per_level`：每升一级所需食物数
- `base_speed_ms`：初始移动间隔
- `speed_step_ms`：每次升级降低的移动间隔
- `min_speed_ms`：速度下限

## 3.2 `SnakeGame`

核心数据：

- `snake`：按顺序保存蛇身体坐标，`snake[0]` 为蛇头
- `direction`：当前移动方向
- `food`：当前食物坐标
- `score`：当前得分
- `foods_eaten`：已吃到的食物数量
- `level`：当前等级
- `speed_ms`：当前移动间隔
- `game_over`：是否结束
- `won`：是否胜利

核心方法：

- `reset()`：重置游戏状态
- `set_direction()`：设置移动方向，并阻止直接反向移动
- `change_direction_by_key()`：将按键映射为方向
- `tick()`：执行一个游戏时钟周期，并返回本次状态结果

## 3.3 `SnakeTerminalApp`

核心能力：

- `render_board()`：把当前游戏状态渲染为文本棋盘
- `direction_from_key()`：将终端按键映射为方向
- `run()`：启动终端交互循环
- 支持 `Q` 退出、`P`/`Space` 暂停、`Enter` 开始、`R` 重开

## 3.4 `ScoreStorage`

核心能力：

- `load_best_score()`：加载历史最高分
- `save_best_score()`：当分数更高时覆盖持久化结果
- `load_leaderboard()`：读取排行榜，可按难度过滤
- `record_run()`：记录完整对局结果并维护排序

## 3.5 `BrowserSnakeController`

核心能力：

- 维护当前选中的难度和正在进行的对局
- 将浏览器动作映射为游戏控制指令
- 负责开始、暂停、重置、难度切换和事件广播
- 为前端页面提供状态快照和排行榜数据

## 4. 关键流程设计

### 4.1 初始化流程

1. 创建 `SnakeGame`
2. 根据配置生成初始蛇身
3. 随机生成不与蛇重叠的食物
4. 将状态渲染到界面

### 4.2 移动流程

1. 根据当前方向计算新的蛇头坐标
2. 判断是否即将吃到食物
3. 检测墙体碰撞和自身碰撞
4. 未碰撞则更新蛇身
5. 如果吃到食物则加分并刷新食物
6. 根据累计食物数更新等级和速度
7. 如果没有空余格子则判定胜利

### 4.3 输入控制流程

1. 监听键盘事件
2. 若为方向键，则更新方向
3. 首次方向输入时开始游戏
4. 若为 `Space`，切换暂停状态
5. 若为 `R`，重置游戏
6. 若在开始菜单中切换难度，则重建待机棋盘配置

### 4.4 运行模式流程

入口文件：`main.py`

运行策略：

- 默认使用终端模式，兼容性最好
- 通过 `python3 main.py --mode gui` 启动图形界面模式
- 通过命令行参数统一管理启动方式
- 两种模式共享同一套等级、分数和最高分逻辑
- 浏览器图形模式额外提供菜单、音效和排行榜体验

## 5. 碰撞与边界策略

本项目采用以下规则：

- 撞墙即失败
- 撞到自身身体即失败
- 不允许直接朝相反方向移动，避免一步撞回自己
- 当蛇未增长时，允许下一步移动到当前尾巴所在位置，因为尾巴会同步离开
- 速度存在下限，避免后期难度失控到不可玩

## 6. 测试设计

测试文件：`tests/test_engine.py`

覆盖场景：

- 初始化是否正确
- 普通移动是否正确
- 吃食物后长度和分数是否正确
- 非法反向移动是否被阻止
- 合法转向是否生效
- 撞墙是否结束
- 撞自己是否结束
- 升级后速度是否正确变化
- 速度是否正确受最小阈值限制
- 移动到旧尾巴位置是否合法
- 棋盘填满后是否正确判定胜利
- 重置后状态是否恢复
- 非法配置是否抛出异常
- 浏览器控制器是否正确切换难度
- 排行榜是否按分数正确排序和过滤
- 终端文本棋盘渲染是否正确
- 终端按键映射是否正确
- 最高分存储在文件缺失或损坏时是否回退安全值

## 7. 可维护性设计

- 核心逻辑与界面完全分离
- 动画表现依赖 `TickResult` 驱动，避免界面层重复推断业务状态
- 浏览器端音效完全在前端合成，不污染核心逻辑
- 通过类型注解提升可读性
- 使用数据类管理配置和状态
- 测试以功能行为为中心，便于回归验证
